Présentation du projet

MD Transition est le site vitrine professionnel de Martine Dorival, facilitatrice et partenaire pour entrepreneurs, artisans, commerçants et associations basée à Conflans-Sainte-Honorine. Le site propose une présentation des services, un formulaire de contact avec envoi d'emails automatisé, et une prise de rendez-vous.

Objectifs techniques

  • Site 100 % statique généré côté serveur pour des performances maximales
  • SEO complet : métadonnées Open Graph, JSON-LD, sitemap automatique
  • Formulaire de contact sécurisé avec protection anti-spam (honeypot) et limitation de débit
  • Envoi d'emails transactionnels via serveur SMTP OVH
  • Design responsive adapté mobile, tablette et desktop
  • Déploiement continu sur Vercel

URL de production

Accès

Site live : https://md-transition.fr
Hébergement : Vercel (serverless)
DNS et messagerie : OVH

Note sur l'architecture

Le projet utilise le App Router de Next.js 14 (dossier src/app/), la convention moderne remplaçant le Pages Router. Les routes API (comme l'endpoint de contact) sont des Route Handlers serverless, exécutés uniquement côté serveur.

Le fichier .gitignore — rôle et contenu

Le fichier .gitignore est une liste de patterns indiquant à Git quels fichiers ou dossiers ne doivent jamais être inclus dans le dépôt. Tout ce qui est dans ce fichier reste uniquement sur la machine locale et n'est ni committé ni poussé vers GitHub ou Vercel.

Contenu et justification des entrées

EntréeCe que cela exclutPourquoi
node_modules/ Toutes les dépendances installées Poids excessif (centaines de Mo), régénéré par npm install
/.next/ Dossier de build Next.js Généré à chaque build, inutile à versionner
.env Fichier de variables d'environnement Contient des mots de passe et clés SMTP ! Ne jamais committer.
.env.local Variables locales (dev)
.env.*.local Toutes variantes locales
.env.production.local Variables production locales
.env.development.local Variables développement locales
.vercel Config Vercel locale Contient des tokens d'accès Vercel
.idea/ .vscode/ Config IDE (IntelliJ, VS Code) Paramètres personnels qui ne concernent pas le projet
.DS_Store, Thumbs.db Métadonnées macOS/Windows Fichiers système inutiles sur le dépôt
Point de vigilance — envoi vers Vercel (vercel --prod)

Lorsque vous déployez avec vercel --prod ou via un push Git, les fichiers .env.local ne sont jamais envoyés. Les variables d'environnement doivent être configurées manuellement dans le dashboard Vercel (voir Section 9). C'est un comportement voulu et sécurisé.

Configuration de l'envoi d'emails via OVH

L'envoi d'emails est géré par la bibliothèque Nodemailer dans le fichier src/lib/email.ts. Le serveur SMTP utilisé est celui d'OVH, où est hébergée la messagerie du domaine md-transition.fr.

Paramètres SMTP OVH

ParamètreValeurDescription
Hôte SMTPssl0.ovh.netServeur sortant OVH
Port SSL465SSL/TLS natif — recommandé
Port STARTTLS587Alternative si 465 est bloqué
ChiffrementSSL/TLSConnexion chiffrée de bout en bout
AuthentificationLogin + Mot de passeIdentifiants du compte email OVH

Flux d'un envoi de formulaire

  1. L'utilisateur soumet le formulaire sur /contact
  2. Le formulaire envoie une requête POST vers /api/contact
  3. L'API vérifie le rate limit (max 3 req/min par IP) et le champ honeypot (anti-spam)
  4. Les données sont validées avec le schéma Zod et sanitisées (protection XSS)
  5. Nodemailer se connecte à ssl0.ovh.net:465 et envoie 2 emails simultanément :
Email 1 — Confirmation au visiteur

Envoyé à l'adresse du visiteur. Confirme la réception du message, rappelle le type de besoin et le texte envoyé.

Email 2 — Notification à l'administrateur

Envoyé à ADMIN_EMAIL. Tableau récapitulatif complet : nom, email, téléphone, type de besoin, message, date/heure Paris.

Créer un compte email chez OVH

  1. Se connecter sur ovh.com/manager
  2. Aller dans Web Cloud → Emails → md-transition.fr
  3. Onglet Comptes email → Créer un compte
  4. Choisir le nom (ex: contact), définir le mot de passe
  5. Reporter ce mot de passe dans .env.local (SMTP_PASS) et dans Vercel

Changer l'adresse email administrateur

L'email administrateur est l'adresse qui reçoit toutes les notifications lorsqu'un visiteur remplit le formulaire de contact. Il est configuré en 3 endroits distincts.

Étape 1 — Modifier le fichier .env.local (environnement local)

Ouvrir .env.local à la racine du projet et changer la ligne :

# Avant ADMIN_EMAIL=test@md-transition.fr # Après (exemple) ADMIN_EMAIL=contact@md-transition.fr
Important

Cette modification ne s'applique qu'en local. Pour la production, il faut aussi mettre à jour Vercel (voir Étape 2).

Étape 2 — Mettre à jour la variable sur Vercel (production)

  • 1
    Aller sur vercel.com

    Se connecter → sélectionner le projet md-transition

  • 2
    Accéder aux variables d'environnement

    Onglet Settings → section Environment Variables

  • 3
    Modifier ADMIN_EMAIL

    Trouver la variable ADMIN_EMAIL → cliquer sur le crayon d'édition → entrer la nouvelle adresse → Save

  • 4
    Redéployer

    Les variables ne sont prises en compte qu'au prochain déploiement. Lancer vercel --prod dans le terminal ou cliquer Redeploy dans Vercel.

Récapitulatif des fichiers impactés

Fichier / EmplacementVariable à changerImpact
.env.local ADMIN_EMAIL=... Environnement local uniquement
Vercel Dashboard → Settings → Env Vars ADMIN_EMAIL Environnement de production

Déploiement local — du clone au navigateur

Cette section décrit la procédure complète pour installer et lancer le projet en local, depuis la récupération du code source jusqu'à l'accès dans le navigateur.

Prérequis

OutilVersion minimumInstallation
Node.js18.x ou supérieurenodejs.org

Étapes d'installation

  • 1
    Cloner le dépôt GitHub
    git clone https://github.com/mdtransitionfr/md-transition.git cd md-transition
  • 2
    Installer les dépendances
    npm install # Crée le dossier node_modules/ (~500 Mo)

    Créer le fichier .env.local

    Lancer le serveur de développement
    npm run dev
  • 5
    Ouvrir dans le navigateur
    http://localhost:3000

npm run build # Build de production (vérification complète) npm run start # Lancer le build de production en local npm run lint # Vérification ESLint du code vercel --prod # Déployer en production sur Vercel

Mise en production sur Vercel

Vercel est la plateforme de déploiement utilisée pour héberger le site. Le déploiement est automatique à chaque push sur la branche main de GitHub, ou manuel via la commande vercel --prod.

Configurer les variables d'environnement sur Vercel

Cette étape est obligatoire au premier déploiement ou après tout changement de configuration.

  • 1
    Se connecter à Vercel

    Aller sur vercel.com → sélectionner le projet md-transition

  • 2
    Accéder aux réglages

    Onglet Settings dans le menu du projet → section Environment Variables

  • 3
    Ajouter chaque variable

    Pour chaque variable : saisir le nom, la valeur, sélectionner l'environnement Production, puis Save.

  • 4
    Redéployer pour appliquer
    vercel --prod # ou : Deployments → Redeploy dans l'interface Vercel

Variables à configurer sur Vercel (production)

SMTP_HOST=ssl0.ovh.net SMTP_PORT=465 SMTP_SECURE=true SMTP_USER=contact@md-transition.fr # Adresse production SMTP_PASS=votre_mot_de_passe_ovh SMTP_FROM=contact@md-transition.fr ADMIN_EMAIL=contact@md-transition.fr # Destinataire des contacts NEXT_PUBLIC_SITE_URL=https://md-transition.fr
Vérification post-déploiement

Après déploiement, tester le formulaire sur https://md-transition.fr/contact. En cas d'erreur 500, consulter les logs dans Vercel → Deployments → votre déploiement → Functions → /api/contact.
