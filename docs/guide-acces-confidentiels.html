Avertissements de sécurité

⛔
Document strictement confidentiel

Ce document contient des informations d'authentification sensibles. Il ne doit être conservé que sur un support sécurisé (coffre-fort numérique, gestionnaire de mots de passe chiffré) et ne doit en aucun cas être partagé par email, messagerie instantanée ou stockage cloud non chiffré.

⚠️
Rotation des mots de passe recommandée

Il est recommandé de changer tous les mots de passe listés dans ce document après toute intervention d'un prestataire externe, ou au minimum tous les 6 mois. Mettre à jour ce document immédiatement après chaque changement.

ℹ️
Utilisation d'un gestionnaire de mots de passe

L'utilisation d'un gestionnaire de mots de passe tel que Bitwarden (gratuit), 1Password ou Dashlane est fortement recommandée pour stocker ces informations de manière sécurisée et éviter de noter les mots de passe en clair.

Procédures
⚙️

Changer l'email administrateur du site

L'email administrateur est l'adresse qui reçoit toutes les notifications du formulaire de contact. Il est configuré via la variable ADMIN_EMAIL. La modification doit être faite à deux endroits : en local ET sur Vercel.

Modification en local (.env.local)

  1. 1
    Ouvrir le fichier .env.local

    À la racine du projet md-projet/

  2. 2
    Modifier la ligne ADMIN_EMAIL

    ADMIN_EMAIL=nouvelle-adresse@domaine.fr

  3. 3
    Redémarrer le serveur local

    Arrêter avec Ctrl+C puis relancer npm run dev

Modification en production (Vercel Dashboard)

  1. 1
    Se connecter sur vercel.com

    Via le compte Google mdtransitionfr@gmail.com

  2. 2
    Projet → Settings → Environment Variables

    Trouver la variable ADMIN_EMAIL

  3. 3
    Modifier et sauvegarder

    Cliquer sur le crayon ✏️ → entrer la nouvelle adresse → Save

  4. 4
    Redéployer

    vercel --prod ou bouton Redeploy dans Vercel

Procédures de changement de mot de passe par plateforme

Plateforme Procédure de changement Impact après changement
Compte Google myaccount.google.com → Sécurité → Mot de passe ⚠️ Vercel et GitHub se déconnectent — reconnecter via le nouveau mot de passe
Vercel Connexion via Google OAuth — pas de mot de passe propre Changer le mot de passe Google suffit
GitHub Connexion via Google OAuth — pas de mot de passe propre Changer le mot de passe Google suffit
OVH Manager ovh.com/manager → Mon compte → Sécurité → Changer le mot de passe Mettre à jour ce document
Email test@md-transition.fr OVH Manager → Emails → Comptes email → Changer le mot de passe ⚠️ Mettre à jour SMTP_PASS dans .env.local ET dans Vercel, puis redéployer
🔑
Après changement du mot de passe SMTP (email OVH)

Si le mot de passe de test@md-transition.fr est changé, il faut impérativement mettre à jour la variable SMTP_PASS dans deux endroits : le fichier .env.local (local) et le dashboard Vercel (production). Sans cette mise à jour, l'envoi d'emails sera cassé en production.
